# 캘린더 상세 영역 공간 부족 — 팀 회의 결과

> 작성일: 2026-03-10
> 참석: 지민(PD), 현우(기획), 소연(비즈니스), 태준(백엔드), 미래(프론트), 윤서(QA), 다은(UI), 승호(UX)

---

## 문제 정의

모바일(390px 기준)에서 캘린더 그리드가 화면의 ~80%를 차지하여,
날짜 클릭 시 하단 상세 영역(DaySheet)이 거의 보이지 않음.

### 수치 분석
- 헤더 + 요약 스트립 + 오늘 스트립: ~200px
- 6주 캘린더 그리드 (셀 min-height 62px): ~372px
- 하단 바: ~50px
- **합계: ~622px** → 대부분 뷰포트 초과
- 상세 영역 가용 공간: 거의 없음 (스크롤 필수)

### 영향
- 핵심 태스크(날짜 선택 → 상세 확인/수정) 완료율 저하
- 첫 사용자 이탈 리스크
- 리텐션 직결 (기록 확인 불가 → 입력 중단)

---

## 후보 방안

### A안: 바텀시트 오버레이

날짜 클릭 시 DaySheet가 캘린더 **위**에 슬라이드업 오버레이로 표시.

| 항목 | 내용 |
|------|------|
| 공수 | 0.5~1일 |
| 장점 | 캘린더 그리드 불변, 빠른 구현, CLAUDE.md 스펙 부합 |
| 단점 | 캘린더가 가려져 날짜 간 빠른 전환 불편 |
| 지지 | 지민, 태준, 다은 |

구현 상세:
- fixed overlay + slide-up transition
- 콘텐츠 양에 따라 가변 높이 (compact 30% / medium 50~60%)
- 오버레이 클릭 또는 하단 드래그로 닫기
- RecordModal과 동일 BottomSheet 래퍼 재활용 가능

### B안: 주간 접기 (Week Collapse)

날짜 클릭 시 선택된 주(1행)만 남기고 나머지 주를 접음.

| 항목 | 내용 |
|------|------|
| 공수 | 1.5~2일 |
| 장점 | 컨텍스트 유지, iOS 캘린더 검증 패턴, 상세 60~70% 확보 |
| 단점 | 애니메이션 복잡도 높음, 엣지케이스 다수 |
| 지지 | 현우, 미래, 승호 |

구현 상세:
- `selectedDate !== null` → 해당 주만 표시
- 나머지 주: `height:0; overflow:hidden; opacity:0` transition (300ms)
- 월 이동 시 자동 복원 (`selectedDate = null`)
- 접힌 상태에서 좌우 스와이프로 같은 주 내 날짜 이동 (선택)

### C안: 셀 높이 축소 (기각)

min-height 62px → 48px 등으로 축소.

| 항목 | 내용 |
|------|------|
| 공수 | 0.5일 미만 |
| 기각 사유 | ~84px만 확보, 이벤트 바 3줄 가독성 저하, 터치 타겟 축소, 근본 해결 안 됨 |

## 합의 사항

1. **우선순위 최상위** — MVP/릴리즈 전 필수 해결
2. **API 변경 불필요** — 순수 프론트엔드 작업
3. **별도 페이지 전환 반대** — 캘린더+상세 같은 화면 유지
4. **C안 기각** — 전원 동의

---

## 미결정 사항

- A안 vs B안 최종 선택 (프로토타입 비교 후 결정)
- RecordModal 표시 방식 (풀스크린 오버레이 유지 vs 인라인)
- DaySheet/RecordModal 상태 관리 구조 (store 통합 vs 독립)

## Definition of Done (QA 기준)

1. iPhone SE, iPhone 15, Galaxy S24, iPad Mini 최소 4개 뷰포트에서 상세 영역 핵심 정보 100% 가시
2. 브라우저 주소창 표시/숨김 양쪽 상태에서 레이아웃 정상
3. 날짜 클릭 → DaySheet → RecordModal → 저장 → 캘린더 반영 전체 플로우 정상
4. 5열 월과 6열 월 양쪽에서 레이아웃 확인
5. 테마 6종 x 배경 4종 중 최소 3개 조합에서 가독성 확인
